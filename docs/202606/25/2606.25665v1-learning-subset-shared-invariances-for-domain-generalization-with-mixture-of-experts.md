## 摘要
领域泛化(DG)旨在从一个或多个源域中学习模型，使其泛化到未见过的目标域，而训练过程中不访问目标域数据。一种常见方法要求所有源域的表征具有不变性，假设预测结构是全局共享的。然而，我们证明，强制更多域的不变性会逐渐限制可行的表征空间，从而丢弃那些并非普遍共享的可迁移预测因子。为解决此限制，我们提出子集共享不变性，即预测结构仅假设在域子集内稳定。我们通过混合专家架构实现这一原则，其中每位专家对齐其所服务的特定域，路由机制组合子集不变组件进行预测。这创建了一种路由条件的不变性，与表征共同学习。为促进有效分解，我们设计了训练目标，鼓励选择性对齐、自信且平衡的路由以及多样化的专家专业化。在DomainBed基准上的实验表明，随着域异质性增加，我们的方法在跨域泛化上表现更优且鲁棒性更强。我们的结果表明，DG应超越强制单一全局不变性，转而通过域子集间的部分共享结构来建模不变性。

## Abstract
Domain generalization (DG) aims to learn a model from one or more source domains that generalizes to an unseen target domain without accessing target data during training. A common approach enforces invariance of representations across all source domains, assuming predictive structure is globally shared. However, we demonstrate that enforcing invariance across more domains gradually restricts the feasible representation space, discarding transferable predictive factors that are not universally shared. To address this limitation, we propose subset-shared invariance, where predictive structure is assumed stable only within domain subsets. We implement this principle with a mixture-of-experts architecture, where each expert aligns the specific domains it serves and a routing mechanism composes subset-invariant components for prediction. This creates a routing-conditioned invariance, jointly learned with the representation. To facilitate effective decomposition, we develop training objectives that encourage selective alignment, confident and balanced routing, and diverse expert specialization. Experiments on DomainBed benchmarks demonstrate improved out-of-domain generalization and greater robustness under increasing domain heterogeneity. Our results suggest that DG should move beyond enforcing a single global invariance and instead model invariance through partially shared structure across domain subsets.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：领域泛化（DG）旨在从多个源域学习模型，使其能泛化到未见过的目标域。传统方法强制所有源域共享一个全局不变表征，认为预测结构是全局共享的。
- **背景**：作者发现，随着源域数量增加或域异质性增强，强制全局不变性会逐渐限制表征空间，丢弃那些仅在部分域子集中稳定的可迁移预测因子。这导致性能退化（如图1所示）。
- **核心洞察**：预测结构并非普遍共享，而是分布在域子集之间。因此，DG 应建模“子集共享不变性”，而非单一全局不变性。

## 2. 方法论：核心思想、关键技术细节与算法流程

- **核心思想**：提出子集共享不变性（Subset-Shared Invariance），即认为不变性仅存在于部分域子集内。通过混合专家（MoE）架构实现：每个专家负责对齐特定域子集，路由机制动态组合各专家的子集不变组件进行预测。
- **关键技术细节**：
  - **架构**：使用预训练 DeiT 作为共享编码器，后接 M=6 个专家（两层 MLP）和一个路由器。路由器根据输入产出专家权重 π(x)。
  - **路由条件子集对齐损失（L_ssi）**：对于每个专家 m、类别 c、域对 (i,j)，计算路由责任 ρ(m)_{k,c}，然后加权 OT 距离或 MMD 距离，仅对齐高责任域对。
  - **专家多样化损失（L_div）**：最小化不同专家输出特征矩阵间的余弦相似度，防止专家退化。
  - **路由稀疏与平衡损失（L_sp、L_bal）**：L_sp 鼓励每个样本路由稀疏（低熵），L_bal 鼓励整体负载平衡。
  - **总损失**：L = L_cls + λ_ssi L_OT_ssi + λ_sp L_sp + λ_bal L_bal + λ_div L_div。
- **算法流程**：
  1. 输入 x 经过共享编码器得到 u。
  2. 路由器输出 π(x)。
  3. 每个专家输出 z^{(m)} = h_m(u)。
  4. 最终表征为 z = Σ π_m z^{(m)}，用于分类。
  5. 在训练时，计算分类损失、子集对齐损失（基于路由责任加权的 OT/MMD）、路由正则项和多样化损失，联合优化。

## 6. 主要结论与发现

1. 全局不变性在域增多时会导致预测信息损失；子集共享不变性可保留更多可迁移因子。
2. MESSI 在 DomainBed 上平均 OOD 准确率达 66.2%（DeiT-S/16），优于 GMoE、OMoE 等强基线。
3. 在域增长实验中，MESSI 的峰值准确率（75.31%）和最终准确率（73.45%）均高于所有 CORAL 变体，且下降幅度最小（1.86点 vs 2.58~7.91点），说明对异质性更鲁棒。
4. 路由学习到了有意义的子集结构，与全局对齐模式显著不同（Jaccard 重叠仅0.04~0.14），且所选槽的对齐误差更低。
5. 组件分析证实：子集对齐损失是主要增益来源，路由稀疏、负载平衡和多样性正则化协同工作。

## 7. 优点

- **理论贡献**：严格分析了全局不变性导致表征信息单调收缩，并引入子集共享不变性形式化定义。
- **方法创新**：将路由机制与条件不变性结合，实现自适应、数据驱动的子集划分，无需预定义子集标签。
- **实验全面**：在四个标准 DG 基准、人工诊断基准和真实大规模场景下验证，消融和诊断实验深入。

## 8. 不足与局限

- **计算开销**：最优版本（OT）训练成本高（约12倍 ERM），MMD 版本虽有改善但仍比 GMoE 高。这可能限制在资源受限环境下的应用。
- **对路由依赖性强**：当源域少、类别覆盖不均衡时，路由可能难以学习有效的子集结构，导致对齐信号噪声。
- **可识别性问题**：路由诱导的潜在子集不可唯一标识，不一定对应语义有意义的域分组。
- **实验限制**：WILDS-iWildCam 使用较低分辨率（224），并非严格复现官方领先管线，绝对分数不宜直接比较。此外，未在更多大规模多域场景（如 WILDS-FMoW）上验证。
- **应用限制**：方法假设部分共享结构存在；若所有域确实具有强全局不变性，则路由分解可能引入冗余，降低统计效率。
