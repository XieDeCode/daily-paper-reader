## 摘要
在遥感目标检测中，卷积神经网络（CNN）擅长捕捉局部细节，而视觉Transformer（ViT）更擅长全局上下文建模。然而，现有的检测器通常依赖于单一的固定骨干网络或手动设计的混合架构，因此无法根据输入的不同复杂度自适应地利用这些互补优势。为了解决这一限制，我们提出了基于强化学习的骨干模块组合（BMCR）。BMCR从现成的CNN和ViT骨干网络中分解出的可复用模块中动态组装输入自适应的推理路径。为了实现这种跨家族组合，我们首先构建了一个可扩展的模块工具箱。具体地，我们将代表性的CNN和ViT骨干网络分解为可复用的功能模块，并用明确的结构、语义和计算元数据封装每个模块，以实现兼容性感知的组装。为了弥合基于网格的CNN特征和基于标记的ViT表示之间的差距，我们设计了一个轻量级的基于最优传输（OT）的过渡接口，确保在保持空间一致性的同时进行分布感知的对齐。然后，骨干组合过程被表述为一个序列决策问题，其中策略网络根据中间的多尺度观测逐步选择与任务相关的模块。为了稳定可复用模块和路由策略的联合优化，我们进一步开发了一种自适应模块协同优化（AMCO）策略，该策略在训练过程中协调模块更新、路由探索和奖励分配。在DOTA-v1.0、DOTA-v1.5和DIOR-R上，BMCR分别实现了79.31%、73.41%和71.86%的mAP，在保持竞争力效率的同时，超越了强大的静态和动态基线最多2.5个百分点。

## Abstract
In remote sensing object detection, Convolutional Neural Networks (CNNs) excel at capturing local details while Vision Transformers (ViTs) are better at global context modeling. However, existing detectors typically rely on a single fixed backbone or a manually designed hybrid architecture, and thus fail to adaptively exploit these complementary strengths across inputs of diverse complexity. To address this limitation, we propose Backbone Module Composition via Reinforcement Learning (BMCR). BMCR dynamically assembles input-adaptive inference paths from reusable modules decomposed from off-the-shelf CNN and ViT backbones. To enable such cross-family composition, we first construct an extensible module toolbox. Specifically, we decompose representative CNN and ViT backbones into reusable functional modules and encapsulate each module with explicit structural, semantic, and computational metadata for compatibility-aware assembly. To bridge the gap between grid-based CNN features and token-based ViT representations, we design a lightweight Optimal Transport (OT) based transition interface that ensures distribution-aware alignment while respecting spatial consistency. The backbone composition process is then formulated as a sequential decision problem, in which a policy network progressively selects task-relevant modules according to intermediate multi-scale observations. To stabilize the joint optimization of reusable modules and the routing policy, we further develop an Adaptive Module Cooperative Optimization (AMCO) strategy that coordinates module updating, routing exploration, and reward assignment during training. On DOTA-v1.0, DOTA-v1.5 and DIOR-R, BMCR achieves 79.31\%, 73.41\% and 71.86\% mAP, respectively, surpassing strong static and dynamic baselines by up to 2.5 points while maintaining competitive efficiency.

- **含义**：作者提出 **BMCR（Backbone Module Composition via Reinforcement Learning）**，通过将现成的 CNN 和 ViT 骨干分解为可复用模块，并利用强化学习（RL）动态组装输入自适应的推理路径，以期在性能和效率之间取得更好的平衡。

## 2. 方法论
- **核心思想**：将骨干网络构建视为一个马尔可夫决策过程（MDP），由一个路由策略网络逐步从异构工具箱中选择特征提取模块，并自动调用接口进行特征过渡。
- **关键技术细节**：
  1. **异构模块工具箱**：从 ResNet、ViT、Swin Transformer、SwinV2、ViTAE、ViTAE-RVSA 等骨干中提取模块（如残差块、Transformer 块），并封装结构、语义、计算元数据。
  2. **最优传输（OT）接口**：针对 CNN 网格特征与 ViT 令牌特征之间的格式和分布差异，设计轻量 OT 过渡层，通过空间正则化的运输成本保持几何一致性，实现分布感知对齐（含 Sinkhorn 求解）。
  3. **序列决策与状态空间**：当前特征经多尺度金字塔观测（MSP）、历史动作嵌入、辅助物体先验拼接后形成状态，策略网络输出模块选择（含终止动作）。动作空间通过元数据掩码排除无效组合。
  4. **AMCO 训练策略**：两阶段优化——① **随机路径预热**（前 N_rand 轮）：冻结策略，均匀采样合法路径，优化模块和检测头，同时训练价值网络（监督辅助质量分数）；② **PPO 路由优化**：启用策略梯度更新，并渐进退火辅助先验和过渡系数，避免离散策略与连续特征学习互相干扰。
- **算法流程简述**：
  - 图像经 stem 得到初始特征 x₀。
  - 每步 t，观察状态 s_t，从掩码动作集采样 a_t，自动确定过渡接口（同族用 Reduction，跨族用 OT），执行模块得到 x_{t+1}。
  - 直到终止或达到最大步数 T_max=12，产生样本特异轨迹。
  - 路由后的多尺度特征经适配器投影到统一通道，送入 FPN 和检测头输出。

## 6. 主要结论与发现
- **性能优势**：BMCR 在所有四个数据集的 SS 和 MS 设置下均取得最高 mAP，最高超越强基线 2.5 点，同时保持 63.9 FPS（1024×1024）的竞争力。
- **架构组合价值**：同时包含 CNN 和 ViT 模块的工具箱比单一类型工具箱提升 3.66~7.59 个点，证明跨家族互补有效。
- **关键组件有效**：OT 接口（+4.71）、MSP 观测（+2.55）、随机预热（+6.61）、辅助先验（+1.28）、辅助检测头（+2.03）均显著影响最终性能。
- **自适应计算分配**：在大规模图像上，BMCR 针对简单场景（稀疏目标）推理时间减少 67.2%，平均减少约 45%，说明计算能按场景复杂度动态调整。
- **训练代价可接受**：虽然训练成本高于固定骨干，但微调代价大幅降低，且推理时仅激活单一路径，内存与固定骨干相当。

## 7. 优点
- **方法论创新**：首次将跨 CNN/ViT 的动态模块组合引入遥感目标检测，通过 RL 实现输入自适应。
- **接口设计**：OT-based 过渡界面理论优雅，可保持空间一致性，优于简单插值或对齐。
- **训练稳定性**：AMCO 通过随机预热、辅助监督、渐进退火避免冷启动和梯度干扰，缓解 RL 在大型组合空间中的训练困难。
- **推理高效**：虽然增加了决策开销（~3.1 ms），但全流程平均 16.4 ms/patch，接近轻量固定骨干。
- **可扩展性**：工具箱支持新增模块注册；预训练模型可微调复用，降低下游适配成本。

## 8. 不足与局限
- **训练成本高**：完整 150 epoch 需要约 48.5 h（6×RTX 6000），对普通团队不友好，且 RL 训练不如监督学习稳定。
- **工具箱依赖**：当前模块仅来源于少数几个预定义骨干，最优点受限于候选集合；未见增量注册或自动扩展机制的详细探讨。
- **缺少跨任务验证**：实验仅覆盖旋转目标检测，未在语义分割、场景分类等任务上验证 BMCR 的通用性。
- **记忆与分析**：峰值内存 46.2 GB 限制使用低显存显卡；未比较与轻量动态网络（如 early exit）的详细内存差异。
- **潜在偏差**：消融实验仅在 DIOR-R 上进行，可能未充分反映在其他数据集上的行为差异；奖励函数中的超参数（λ_cost, λ_sw 等）在不同场景下可能需要手动调整。
