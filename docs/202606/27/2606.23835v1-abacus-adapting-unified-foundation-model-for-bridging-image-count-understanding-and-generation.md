## 摘要
ABACUS是一个统一的视觉语言模型，能够处理物体计数、人群计数、指代表达计数以及忠实于计数的图像生成，而无需任何特定于基准的训练。我们的模型基于现有的30亿参数统一基础模型，并通过三项关键创新适配于目标定位任务：利用物体性图进行密度感知自适应缩放以实现空间定位；通过GRPO实现边界感知计数策略以消除裁剪边界误差；以及一种循环一致的GRPO策略，其中理解分支自我批评生成的输出，无需任何外部标注即可弥合理解与生成之间的差距。ABACUS在七个基准测试中均取得了最先进的结果，超越了特定任务专用模型和更大的通用模型。

## Abstract
ABACUS is a unified vision-language model that handles object counting, crowd counting, referring-expression counting, and count-faithful image generation without any benchmark-specific training required. Our model is built on existing 3B-parameter unified foundation model and is adapted for object localization tasks using three key innovations: density-aware adaptive zooming with objectness maps for spatial grounding; a boundary-aware count policy via GRPO to eliminate crop-boundary errors; and a cycle-consistent GRPO strategy where the understanding branch self-critiques generated outputs, closing the understanding-generation gap without any external annotations. ABACUS achieves state-of-the-art results across seven benchmarks, outperforming both task-specific specialists and larger generalist models.

# ABACUS 论文总结

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **问题**：现有的统一视觉语言模型（如 UniLIP、BAGEL）虽然能同时处理理解与生成任务，但在**密集场景下的计数**和**忠实于计数的图像生成**两方面表现不佳，且两者之间存在显著的“协同差距”（synergy gap）——同一个模型能正确统计图像中 4 个苹果，却无法从文本提示生成恰好 4 个苹果的图像。
- **背景**：物体计数、人群计数、指代表达计数和计数条件图像生成长期以来被当作独立任务，各有专用架构和训练流程，缺乏统一、零样本可用的模型。虽然近年出现了一些统一多模态模型（UMM），但计数任务要求精确的实例级空间推理，而纯生成目标无法强制正确数量，导致理解与生成相互掣肘。
- **论文含义**：提出 ABACUS，首次在单一 3B 参数模型中同时实现零样本的计数理解和计数忠实生成，证明了理解与生成不是竞争目标，而是可以相互增强的互补任务。

## 2. 论文提出的方法论
- **核心思想**：基于现有统一基础模型（UniLIP-3B），通过三项关键创新轻量适配，无需任何基准专用训练即可统一处理多种计数任务和生成任务。
- **关键技术细节**：
  - **密度感知自适应缩放（Density-Aware Adaptive Zooming）**：利用冻结的 GroundingDINO + 轻量 MLP 作为密度指示器，将密集图像递归划分为 2×2 子区域，直到达到最小分辨率 γ，然后在每个子区域独立进行计数，最后聚合全局计数。这使 MLLM 在局部稀疏区域更可靠。
  - **物体性图注入（Infusing Objectness in MLLM）**：从 MLLM 的多头自注意力层中提取物体性图。通过逐头隔离和学习的仿射对齐，得到每个视觉 token 位置的空间分布，从中识别物体峰值，并利用高斯平滑的真值点标注进行监督（物体性正则化损失 \(L_{obj}\)），引导模型进行原则性空间推理而非 token 记忆。
  - **边界感知计数策略（Boundary-Aware Count Policy via GRPO）**：自适应划分会在裁剪边界产生物体归属问题。该策略使用 GRPO 后训练，输出结构化的分类结果（内部、边缘、边界），通过三个嵌套奖励（局部准确性、跨象限边界一致性、全局计数连贯性）消除过计数/欠计数错误。
  - **循环一致性 GRPO（Cycle-Consistent GRPO）**：对于生成分支，使用冻结的理解分支作为内置计数模型，对生成的候选图像进行计数偏差评估，并结合美学评分作为复合奖励，仅更新生成分支。这形成闭环：生成→自计数→对比提示→更新生成器，随着生成改善，理解分支的奖励信号也逐步变强，从而实现协同提升。
- **算法流程**（文字说明）：
  1. **理解分支训练**：使用密度自适应缩放后的图像，联合 next-token 预测损失 + 物体性正则化损失微调 MLLM（LoRA），然后通过边界感知 GRPO 后训练（2K步）。
  2. **连接器训练**：冻结理解分支，训练连接器对齐 MLLM 输出特征与 DiT 条件空间。
  3. **生成分支训练**：训练连接器与 DiT 在大规模计数条件数据上，然后通过循环一致性 GRPO（5K步）优化，使用理解分支的计数偏差 + 美学奖励，仅更新生成分支。

## 7. 优点
- **零样本泛化**：无需任何基准专用训练，单个模型即可处理多种计数和生成任务，实际应用友好。
- **理解与生成闭环**：利用理解分支自评生成结果，无需外部标注或批评器，实现真正的端到端相互增强。
- **轻量高效**：仅微调 1.6% 参数（LoRA），训练总时长 44 小时，算力开销远低于主流大规模预训练。
- **设计巧妙**：密度自适应缩放 + 物体性图提供空间推理基础；边界感知 GRPO 以奖励函数形式解决裁剪误差，避免复杂后处理；循环 GRPO 稳定生成优化。
- **实验全面且公平**：覆盖 7 个主流基准，多种消融，人工评估验证用户偏好，且严格控制数据泄漏。

## 8. 不足与局限
- **低分辨率/压缩输入性能下降**：依赖 14×14 patch token，对 <224px 或重度压缩图像（如监控、老旧数据集）效果差，需超分辨率预处理。
- **领域分布偏移**：在医学细胞、卫星车辆、工业缺陷检测等专业领域未见训练，当前计数平衡训练混合物未覆盖，需轻量域适配。
- **密度极高场景推理延迟**：自适应缩放最多触发 3 级递归，最坏情况下延迟随子区域数增长（但实际 <3% 场景触发，且收益远大于代价）。可考虑早退策略。
- **单模型通用性 vs 专业精度**：尽管 SOTA，但专用双阶段模型在某些高度专业化任务上仍有潜力（如表 2 中 GrREC 的 MAE 6.50 略优于 ABACUS 7.67），ABACUS 通过统一模型牺牲了极个别场景的极致性能。
- **缺乏更广泛的空间推理验证**：作者指出循环自奖励范式可推广到其他空间推理任务，但本文仅验证了计数任务。
