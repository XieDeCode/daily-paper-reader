## 摘要
暗船检测需要融合船舶通过AIS报告的信息与卫星通过雷达和光学传感器观测到的数据。DarkVesselNet是一个多模态遥感堆栈，结合了Sentinel-1 SAR、Sentinel-2光学影像、地理空间基础模型骨干、AIS轨迹推理、TGARD式间隙检测以及受Pi-DPM启发的异常检测头。该仓库将系统公开为一个经过测试的Python包和一个公开的Hugging Face Space。本文介绍了传感器堆栈、骨干抽象、融合路径、异常检测头以及当前的验证。目前可用的证据是基于软件的：对SAR散斑滤波、光学波段比值、哈弗辛距离、TGARD间隙发射、传感器配准、骨干令牌形状以及可微分异常评分的测试。

## Abstract
Dark vessel detection requires fusing what vessels report through AIS with what satellites observe through radar and optical sensors. DarkVesselNet is a multi-modal remote sensing stack that combines Sentinel-1 SAR, Sentinel-2 optical imagery, geospatial foundation model backbones, AIS trajectory reasoning, TGARD-style gap detection, and a Pi-DPM-inspired anomaly head. The repository exposes the system as a tested Python package and a public Hugging Face Space. The paper presents the sensor stack, backbone abstraction, fusion path, anomaly head, and current validation. The evidence currently available is software-grounded: tests for SAR speckle filtering, optical band ratios, Haversine distance, TGARD gap emission, sensor coregistration, backbone token shapes, and differentiable anomaly scoring.

## 1. 核心问题与整体含义

- **研究动机**：海上领域感知依赖于检测那些停止广播、伪造位置或在AIS数据不可靠区域航行的船舶（即“暗船”）。单一传感器（如SAR、光学或AIS）存在固有缺陷：SAR全天候但受斑点噪声和近岸杂波影响；光学图像日间清晰但受云层和夜间限制；AIS语义丰富但基于合作报告，存在缺失、欺骗或覆盖盲区。
- **整体含义**：暗船检测本质上是一个“证据分歧”问题——当AIS报告、SAR观测、光学影像、轨迹行为等信息流之间出现无法用常规原因解释的不一致时，候选目标才值得注意。系统不应输出单一概率，而应提供可追溯的融合决策，辅助分析师审查，而非自动判定非法活动。

## 2. 方法论

### 核心思想
- 构建一个模块化、可交换骨干的多模态融合堆栈，对不同传感器的观测进行编码，保留可用性掩码，通过注意力机制融合，并输出带有证据跟踪的可解释告警。

### 关键技术细节
- **传感器预处理**：
  - SAR路径：Lee滤波抑制斑点噪声，验证了在恒定图像上的幂等性和对合成斑点的方差降低。
  - 光学路径：云掩膜与波段比值特征（NDVI、NDWI）。
- **地理空间基础模型骨干**：设计统一的`GeoBackbone`适配器，支持Prithvi-2、Clay、SatMAE++、DOFA、SatlasNet、RemoteCLIP等骨干，返回形状为`(B, N, D)`的patch token。在CPU测试中使用轻量级后备投影模拟token输出。
- **AIS轨迹推理**：将AIS窗口表示为`(t, φ, λ, sog, cog)`序列，采用TGARD风格标记长时间或不可达间隙，使用哈弗辛公式计算测地距离。
- **多模态融合**：融合嵌入时保留模态可用性掩码，防止模型将光学缺失误判为暗光学场景。
- **异常检测头**：受Pi-DPM启发，将场景token与AIS段token池化后融合，预测logit和重构的AIS段，设计为轻量级推理头。
- **告警分数分解**：告警分数可分解为`z_sar + z_ais + z_traj + z_opt + z_ctx`，便于审计。
