## 摘要
我们提出MM++（多层马氏距离++），一个完全无监督、严格事后且尺度不变的分布外（OOD）检测框架。为了解决尺度不变性与层次表达性之间的权衡，MM++构建了一个规范化的联合特征空间。它首先通过测量熵密度下降来识别具有判别性的中间层，这些下降标志着语义压缩的边界。通过将这些选定层与终端表示融合，该框架捕捉了潜在的跨层相关性，同时减轻了早期层的噪声。关键的是，Ledoit-Wolf正则化的绑定协方差矩阵稳定了该统一空间，从而实现了可靠的距离估计。无需辅助OOD数据、分类器微调或架构修改，MM++在近端和远端OOD检测的不同架构上均提供了稳健的性能。

## Abstract
We introduce MM++ (Multilayer Mahalanobis++), a fully unsupervised, strictly post-hoc, and scale-invariant framework for Out-of-Distribution (OOD) detection. To address the trade-off between scale invariance and hierarchical expressivity, MM++ constructs a principled joint feature space. It first identifies discriminative intermediate layers by measuring entropy density drops, which mark the boundaries of sharp semantic compression. By fusing these selected layers with the terminal representation, the framework captures latent cross-layer correlations while mitigating early-layer noise. Crucially, a Ledoit-Wolf regularized tied covariance matrix stabilizes this unified space, enabling reliable distance estimation. Requiring no auxiliary OOD data, classifier fine-tuning, or architectural modifications, MM++ delivers robust performance across distinct architectures for both near- and far-OOD detection.

# MM++: 基于Top-K门控特征融合的无监督尺度不变多层OOD检测

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：深度神经网络在开放世界中常对分布外（OOD）输入产生过度自信的预测，这对安全关键应用（如医疗诊断、自动驾驶）构成风险。现有OOD检测方法面临两大挑战：
  - 单层方法（如Mahalanobis++）仅利用终端的倒数第二层表示，受限于“神经崩溃”现象（终端表示高度压缩，丢失了中间层的细粒度结构信息），难以检测近OOD样本（语义相似但分布偏移）。
  - 多层方法（如Mahalanobis、X-Mahalanobis）虽然引入中间层，但通常采用独立分数相加或加权融合，忽略了跨层依赖关系；且X-Mahalanobis需要微调或代理OOD数据，违反了严格事后（post-hoc）和无监督的假设。
- **目标**：提出一个**完全无监督、严格事后、尺度不变**的多层OOD检测框架，在不依赖额外数据或模型修改的前提下，通过建模跨层联合分布来同时捕捉近端和远端OOD信号。

## 2. 论文提出的方法论

### 核心思想
- **MM++** 通过构建一个**统一的联合特征空间**（而非分数相加）来融合多个层次的信息。该空间由选定的重要中间层与终端层（倒数第二层）的ℓ2归一化特征拼接而成，并采用Ledoit-Wolf正则化的绑定协方差矩阵进行稳定估计，最后在该联合空间上计算单一的Mahalanobis++距离作为OOD分数。

### 关键技术细节
1. **层选择：基于熵密度下降的Top-K门控机制**
   - 对每个中间层l，计算其归一化协方差矩阵的Shannon熵 \(H_l\)，再除以该层维度得到熵密度 \(\rho_l = H_l / D_l\)。
   - 定义相邻层的熵密度下降：\(\Delta_l = \rho_{l-1} - \rho_l\)，大的正值表示该层发生了剧烈的语义压缩（信息瓶颈），具有最强的判别性。
   - 始终锚定终端层（L），从中间层中选择\(\Delta_l\)最大的K-1层（要求\(\Delta_l > 0\)），得到选定层集合K。

2. **特征融合**
   - 对每个选定层l，将特征进行ℓ2归一化后，直接拼接成统一向量：\(\phi(x) = [\tilde{h}_{l_1}(x);\ldots;\tilde{h}_{l_K}(x)]\)。
   - 无需显式的层间权重，跨层相互作用由联合精度矩阵隐式学习。

3. **联合协方差估计与正则化**
   - 基于ID训练数据计算融合空间的类条件均值 \(\hat{\mu}_c^K\) 和绑定协方差矩阵 \(\hat{\Sigma}_K\)。
   - 由于拼接后的维度可能很大，直接求逆可能病态；采用 **Ledoit-Wolf收缩** 得到正则化协方差矩阵 \(\hat{\Sigma}_{K,\text{shrink}}\)，保证正定性。
   - 进而计算精度矩阵 \(\hat{\Sigma}_K^{-1}\)。

4. **OOD评分**
   - 测试样本x的OOD分数为联合空间上负的最小马氏距离：\(S_{\text{MM++}}(x) = -\min_c (\phi(x)-\hat{\mu}_c^K)^\top \hat{\Sigma}_K^{-1} (\phi(x)-\hat{\mu}_c^K)\)。
   - 分数越高表示越可能是ID样本。

### 算法流程
- **离线校准阶段**：提取所有ID训练样本的层特征，计算熵密度下降，选定K个层，计算融合类均值和Ledoit-Wolf收缩后的精度矩阵。
- **在线推理阶段**：对测试样本提取选定层的归一化特征并拼接，计算联合马氏距离得分。

## 6. 论文的主要结论与发现

1. **MM++在近OOD检测上显著优于单层方法**：尤其在长尾ID（ImageNet-LT）设置下，MM++在ImageNet-C、ImageNet-ES、ImageNet-R、ImageNet-V2等近OOD基准上获得了最高的AUROC和最低的FPR95（例如在ViT-B/16上平均AUROC达83.91% vs 单层Mahalanobis++的80.22%）。
2. **跨层联合建模比分数相加更有效**：MM++通过显式建模跨层依赖关系，能够检测出在单层边缘上看似ID但联合轨迹异常的高阶OOD样本，而X-Mahalanobis的加权相加无法捕捉这种联合偏差。
3. **Ledoit-Wolf正则化在联合高维空间中至关重要**：使用伪逆估计会导致性能大幅下降（表3：AUROC从84.35%降至76.09%），而收缩保证了数值稳定性和泛化。
4. **MM++具有架构通用性**：在ViT、Swin、ConvNeXt、EVA02等不同设计范式上均能稳定改进近OOD性能，而X-Mahalanobis由于其同质性假设无法直接应用于Swin和ConvNeXt。
5. **K=2提供了最佳效率-性能权衡**：融合终端层和一个信息最丰富的中间层足以达到接近最优的检测性能，增加更多层反而可能引入冗余噪声。

## 7. 优点

1. **完全无监督与事后**：不需要任何辅助OOD数据、不需要模型微调或架构修改，可直接应用于预训练模型。
2. **尺度不变**：通过对所有层特征进行ℓ2归一化，消除了特征范数差异带来的偏差。
3. **信息论驱动的层选择**：利用熵密度下降自动定位语义压缩边界，无需手动设计权重。
4. **联合特征空间建模**：突破传统分数相加的独立性假设，捕捉了跨层依赖关系。
5. **数值稳定**：使用Ledoit-Wolf收缩处理高维协方差估计问题，保证精度矩阵的可逆性和鲁棒性。
6. **高效**：仅引入单个超参数K；在线推理延迟与单层Mahalanobis++相当，远低于X-Mahalanobis。
7. **多架构泛化**：在全局注意力、层次注意力、卷积网络中均表现优异，适用性广泛。

## 8. 不足与局限

1. **依赖假设**：层选择依赖于语义压缩过程中存在显著的熵密度下降；若模型中间层缺乏明确的信息瓶颈结构（如某些强正则化模型），可能无法选出有效层。
2. **计算开销**：虽然在线推理很轻，但离线校准阶段需要为所有训练样本提取多层特征并计算协方差特征分解，对大规模模型（如ViT-L）可能耗时较高（但文中未提供具体数字）。
3. **仅适用于分类任务**：当前方法针对分类头的特征表示设计，未涉及密集预测任务（目标检测、语义分割），扩展到这类任务需要处理空间局部化特征塌陷。
4. **局限性验证**：未评估在非自然图像分布（如医学图像、遥感图像）或对抗攻击下的表现；实验仅在ImageNet系列数据集上进行，可能存在数据集偏差。
5. **性能波动**：当K增加时，AUROC呈非单调变化，表明冗余层可能引入噪声；虽然推荐K=2，但在某些架构上可能不是最优选择。
6. **公平性风险**：作为OOD检测工具，不能保证检测所有未知异常，过度依赖可能导致安全违规；论文建议将其作为更广泛安全框架的一部分。
