## 摘要
天气和气候基础模型产生高维预测，仅凭静态图难以检查其学习到的关系。GeoXplain 是一个基于 Python 的交互式可视化工具包，用于探索跨气候变量、气压层和预报时间的地理空间归因图。该工具包接受包含归因网格及相应元数据的归因包，并在笔记本小部件或浏览器中以地图和全球模式、关联时间线、气压层控制、目标注释以及可选的物理场叠加进行渲染。我们将 GeoXplain 定位为模型无关的地球系统可视化工具包，并展示其首个计算后端 GeoXplain Aurora 适配器。该适配器为 Aurora 基础模型计算解释，可通过本地 GPU 进程、GPU 监听器或基于 SLURM 的监听器执行，同时为分析师保留相同的 Python 调用接口。目前支持梯度显著性、积分梯度、RISE、ViT-CX、多帧显著性和积分梯度展开，以及 ERA5 叠加检索。GeoXplain 可通过 pip install geoxplain 安装为 PyPI 包。代码开源，地址为 https://github.com/clemenskoprolin/geoxplain。

## Abstract
Weather and climate foundation models produce high-dimensional forecasts whose learned relationships are difficult to inspect with static plots alone. GeoXplain is an interactive Python-based visualization toolkit for exploring geospatial attribution maps across climate variables, atmospheric pressure levels, and forecast time. The toolkit accepts attribution bundles containing attribution grids together with corresponding metadata and renders them in a notebook widget or browser with map and globe modes, linked timelines, pressure-level controls, target annotations, and optional physical-field overlays. We frame GeoXplain as a model-agnostic earth-system visualization toolkit and present the GeoXplain Aurora Adapter as its first computation backend. The adapter computes explanations for the Aurora foundation model, either in a local GPU process, through a GPU listener, or through a SLURM-backed listener, while preserving the same Python call site for analysts. It currently supports gradient saliency, Integrated Gradients, RISE, ViT-CX, multi-frame saliency and Integrated Gradients rollouts, and retrieval of ERA5 overlays. GeoXplain can be installed as a PyPI package with pip install geoxplain. The code is open-source and available at https://github.com/clemenskoprolin/geoxplain.

## 1. 论文的核心问题与整体含义（研究动机和背景）
- **研究动机**：天气和气候基础模型（如Aurora、GraphCast等）生成高维、多变量、多压力层、多时间步的预测，其学习到的模式难以通过静态图单独检查。分析人员需要回答可解释性问题（如“哪些输入影响了某地的湿度预报？”、“不同压力层下的归因是否不同？”），但现有工作流通常将探索性分析、批量计算和静态绘图分离，导致工作流程中断、速度慢。
- **核心问题**：如何提供一个交互式、模型无关的可视化工具，使气象科学家能够在计算笔记本内直接、流畅地探索归因图（attribution maps），同时保留地理空间上下文（变量、压力层、预报时间、目标位置）。
- **整体含义**：GeoXplain填补了高维天气模型解释分析与交互式地理空间可视化之间的空白，通过“结果包”格式解耦计算与可视化，支持即时（on-the-fly）解释计算和灵活的交互探索。

## 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程
- **核心思想**：将可视化层与模型特定计算层分离。可视化层（GeoXplain viewer）仅处理标准化结果包，计算层（如Aurora适配器）负责归因计算并返回结果包。用户通过笔记本调用同一Python接口，可选择本地、远程GPU监听或SLURM后端执行。
- **结果包格式**（Result Bundle）：自描述的数据格式，包含归因网格（float32二维数组）、元数据（方法、时间框架、目标、输入变量、垂直层等）。支持内存或文件(.xai.npz/.overlay.npz)存储。
- **目标定义**（Target）：指定输出变量、预报时间、可选压力层、空间区域（点目标或框平均）。框目标输出经纬度矩形内的算术平均。
- **支持的归因方法**（在Aurora适配器中实现）：
  - **Saliency**（显著性）：计算标量目标 \( S(X) = \tau(F(X)) \) 对输入场 \( X_{v,t_1} \) 的梯度：\( A_{sal}^v = \frac{\partial S(X)}{\partial X_{v,t_1}} \)。
  - **Integrated Gradients**（积分梯度，IG）：沿从平滑基线 \( X' \) 到实际输入 \( X \) 的直线路径积分梯度，使用中点采样：\( A_{IG}^v = (X_{v,t_1} - X'_{v,t_1}) \cdot \frac{1}{n} \sum_{k=1}^n \frac{\partial S(X^{(k)})}{\partial X_{v,t_1}^{(k)}} \)，基线通过高斯平滑（σ=2.5°）获得。
  - **RISE**：通过随机掩码混合输入与基线，测量目标变化，使用中心化回归估计器（避免分类RISE的归一化问题），掩码从低分辨率伯努利网格双线性上采样生成，水平掩码在气压层上共享（列式扰动）。
  - **ViT-CX**：针对Aurora分层编码器改造，选择特定编码器阶段（stage 2），对垂直令牌组求平均后聚类空间特征，每个聚类用作遮挡区域，计算遮挡后目标得分变化，最后赋值给网格。
  - **Rollout（展开）**：对多步自回归预报，评估最终目标，计算相对于初始输入状态的Saliency或IG（排除RISE和ViT-CX因计算成本）。
- **适配器数据流**：适配器加载WeatherBench格式案例数据 → 构建Aurora批 → 解析目标为可微标量 → 运行选定方法 → 将输出数组转换为GeoXplain帧（含变量、压力层、色图等元数据） → 打包为结果包。
- **可视化交互**：支持地图（MapLibre）和3D球体（Three.js）模式，提供时间线播放、压力层可见性/透明度控制、热力图/等值线切换、色图选择、目标注释、物理场叠加（ERA5等）。所有图层采用标准化、量化编码为WebGL纹理，通过着色器渲染。
